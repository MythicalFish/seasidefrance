# Visual Style Guide

Use this document for the shared visual language of the project.

## Design Philosophy

Capture the high-level intent of the UI here. For example:

- brand personality
- level of visual density
- accessibility and readability priorities
- motion and interaction tone

## Core Principles

1. Favor consistency over one-off styling decisions.
2. Use reusable tokens and shared patterns before creating new variants.
3. Keep hierarchy clear through spacing, typography, and contrast.
4. Document meaningful exceptions instead of letting them drift into the codebase silently.

## Color Palette

Document the main colors and their intended use:

- surfaces
- text hierarchy
- accents
- borders and dividers
- semantic states such as success, warning, and error

## Typography

Document:

- primary and secondary fonts
- heading and body usage
- scale and weight guidance
- special treatment for labels, captions, and metadata

## Spacing And Radius

Document the spacing scale, layout rhythm, border radius, and any layout grid constraints.

## Key Patterns

Document recurring UI patterns such as:

- cards and panels
- buttons and links
- forms and validation states
- navigation
- modals, drawers, or overlays
